clawlingua(1) -- 从本地文本/EPUB 构建 Anki 填空牌组

用法
    python -m clawlingua.cli 命令 [选项]

说明
    ClawLingua 将真实语料（播客字幕、文章、教材例句）转换为
    Anki 填空牌组（.apkg）。

    输入为本地 .txt/.md/.epub 文件。网页抓取请交给上游工具
    （例如 clawfetch），生成 Markdown 或纯文本后再交给 ClawLingua。

命令
    init
        初始化运行环境。

        - 若不存在 .env，则从 .env.example 复制一份；
        - 校验以下文件是否存在：
          ./prompts/cloze_contextual.json
          ./prompts/cloze_textbook_examples.json
          ./prompts/translate_rewrite.json
          ./templates/anki_cloze_default.json

    doctor
        检查依赖、配置、prompt、LLM 连通性、TTS 和输出目录。

        用法：
            python -m clawlingua.cli doctor [--env-file FILE]

    build deck
        从本地文件构建 Anki .apkg 牌组。

        用法：
            python -m clawlingua.cli build deck INPUT \
              --source-lang CODE \
              --target-lang CODE \
              [--content-profile general|textbook_examples] \
              [--input-char-limit N] \
              [--env-file FILE] \
              [--output PATH] \
              [--deck-name NAME] \
              [--max-chars N] \
              [--cloze-min-chars N] \
              [--max-notes N] \
              [--temperature FLOAT] \
              [--difficulty beginner|intermediate|advanced] \
              [--save-intermediate/--no-save-intermediate] \
              [--continue-on-error] \
              [--debug]

        INPUT
            本地文件路径，支持 .txt/.md/.epub。

        选项
            --source-lang
                覆盖 env 中的源语言（默认从配置读取）。

            --target-lang
                覆盖 env 中的目标语言。

            --content-profile
                内容策略，支持：
                    general
                        通用上下文内容；使用 prompts/cloze_contextual.json。
                    textbook_examples
                        教材例句模式；使用
                        prompts/cloze_textbook_examples.json，并对
                        CLOZE_MIN_CHARS 做额外约束。

            --input-char-limit
                仅处理输入前 N 个字符，便于快速测试（例如
                --input-char-limit 4000）。

            --env-file
                指定 .env 文件路径；省略时使用默认查找逻辑。

            --output
                输出 .apkg 路径。默认 outputs/<run_id>/output.apkg。

            --deck-name
                覆盖默认的牌组名称。默认值为
                CLAWLINGUA_DEFAULT_DECK_NAME 或输入文件名（去掉扩展名）。

            --max-chars
                覆盖本次运行的 CLAWLINGUA_CHUNK_MAX_CHARS。

            --cloze-min-chars
                覆盖本次运行的 CLAWLINGUA_CLOZE_MIN_CHARS。

            --max-notes
                限制导出的总卡片数。

            --temperature
                LLM 温度（覆盖 env）。

            --difficulty
                覆盖 CLAWLINGUA_CLOZE_DIFFICULTY，取值：
                    beginner | intermediate | advanced

            --save-intermediate / --no-save-intermediate
                是否在 CLAWLINGUA_OUTPUT_DIR/<run_id> 下输出
                中间 JSONL 文件（chunks、候选、翻译、最终卡片等）。

            --continue-on-error
                遇到单条 cloze/翻译/TTS 错误时继续执行，并将错误
                记录到 errors.jsonl，而不是整批失败。

            --debug
                出错时抛出完整 traceback，便于调试。

    prompt validate
        校验 prompt JSON 文件是否符合预期 schema。

        用法：
            python -m clawlingua.cli prompt validate PATH

    config show
        以 JSON 形式展示合并后的运行配置。

        用法：
            python -m clawlingua.cli config show [--env-file FILE]

    config validate
        校验配置是否有效但不构建牌组。

        用法：
            python -m clawlingua.cli config validate [--env-file FILE]

环境变量
    CLAWLINGUA_LLM_*             主 LLM 配置（OpenAI-compatible）
    CLAWLINGUA_TRANSLATE_LLM_*   翻译小模型配置（可选）
    CLAWLINGUA_CHUNK_*           切块参数
    CLAWLINGUA_CLOZE_*           cloze 粒度控制
    CLAWLINGUA_CONTENT_PROFILE   默认内容策略
    CLAWLINGUA_TTS_*             TTS 提供方与 voice 列表
    CLAWLINGUA_OUTPUT_DIR        输出目录（中间运行数据 JSONL/媒体）
    CLAWLINGUA_EXPORT_DIR        导出牌组目录
    CLAWLINGUA_LOG_DIR           日志目录
